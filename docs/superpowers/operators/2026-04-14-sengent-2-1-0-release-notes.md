# Sengent 2.1.0 Release Notes

## Summary

Sengent 2.1.0 is the first governance-first hosted-runtime release.

Sengent 1.0 used Ollama as the primary runtime path. Sengent 2.1.0 moves the primary runtime path to an OpenAI-compatible API while preserving the original governance model:

- runtime truth still comes from reviewed active knowledge packs
- hosted factory drafting remains offline and review-only
- clarify-first, boundary pack, tool arbitration, audit trail, and rollback remain in force

## What Is New

- primary runtime path is now `OpenAI-compatible API` based
- hosted factory drafting is available as a review-only path
- outbound trust-boundary hardening covers runtime and chat-polish paths
- provider-neutral request seam is in place for hosted runtime
- adversarial support drill now includes backfilled real customer cases

## What Stayed The Same

- Sengent is still not a raw-doc RAG bot
- raw ingestion still does not become runtime truth
- build / review / gate / activate / rollback still control knowledge lifecycle
- deterministic diagnostics and tool-required boundaries still apply

## Installation Highlights

### Runtime host

```bash
bash scripts/install_sengent.sh
source .venv/bin/activate

export SENGENT_RUNTIME_LLM_PROVIDER=openai_compatible
export SENGENT_RUNTIME_LLM_BASE_URL=https://your-llm-endpoint.example.com
export SENGENT_RUNTIME_LLM_MODEL=your-runtime-model
export SENGENT_RUNTIME_LLM_API_KEY=your-runtime-api-key

sengent doctor
sengent chat
```

### Optional hosted factory drafting

```bash
export SENGENT_FACTORY_HOSTED_PROVIDER=openai_compatible
export SENGENT_FACTORY_HOSTED_BASE_URL=https://your-llm-endpoint.example.com
export SENGENT_FACTORY_HOSTED_MODEL=your-factory-model
export SENGENT_FACTORY_HOSTED_API_KEY=your-factory-api-key
```

### Legacy compatibility

If you still need the old local-model path:

```bash
bash scripts/install_sengent.sh --ensure-ollama-model
```

This remains a compatibility path, not the primary 2.1.0 runtime story.
