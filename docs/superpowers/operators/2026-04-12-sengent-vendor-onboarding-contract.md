# Sengent Vendor Onboarding Contract

## Required Inputs

Every vendor profile must be onboarded with:

- `official sources`
- `domain standards`
- `playbooks`
- `incident cases`
- `eval corpus`
- `support boundaries`

## Acceptance Rule

Do not publish a vendor profile until each required input is represented, reviewed, and traceable to the profile manifest.

## Support Boundary Rule

The profile must state what the support kernel may answer, what it must refuse, and when it must ask for clarification instead of guessing.

## Practical Constraint

If a source is unofficial, incomplete, or outside the support boundary, treat it as supporting context only. It does not become vendor truth until it passes the profile review path.
